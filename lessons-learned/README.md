# Lessons Learned

This directory stores technical lessons that remain useful to current operations.
Old issue-system receipts and role-based governance artifacts are not active policy.

Use [LESSONS_PUBLISH_PROCESS.md](LESSONS_PUBLISH_PROCESS.md) to add or update a
lesson. A lesson must cite the underlying evidence, distinguish observation from
inference, and state the concrete guard or test that changed.

Current authority stays in `CLAUDE.md`, `processes/process_registry.md`, and the
active task-specific runbook. A lesson cannot create an approval layer or override
an OWNER decision, deterministic gate, or exact deployment manifest.
