## CRITICAL FLAG — Zero-Trade Scaffold: DL-054 G4 Will INVALID

**Location:** `Strategy_EntrySignal()`, line 135.

```mq5
return false;   // ← hardcoded, always fires regardless of z-score signal
```

`Strategy_EntrySignal()` computes the z-score and sets `req.reason`, but unconditionally returns `false`. No entries are ever taken in any backtest run. This is stated as intentional P1 scaffold behavior (comment line 125: "P1 scaffold keeps entries disabled until cadf/2-leg executor is wired").

**Impact on DL-054 pipeline:**

DL-054 G4 (`gate4_trade_evidence`) requires ≥1 trade in the backtest, or an explicit zero-trade ADR (Architectural Design Record) on file. Without either:
- G4 will return verdict=INVALID for every symbol run
- Phase verdict = INVALID_GATE_FAILURE (no symbol passes)
- Pipeline cannot advance QM5_1017 through any DL-054-gated phase

This is **not** the same as QUA-662 (phantom-PASS): that was a configuration failure masking real zero-trade errors. This is a by-design zero-trade scaffold that needs explicit pipeline accommodation.

**Required before pipeline dispatch:**

One of:
1. **Zero-trade ADR** — file a `zero_trade_ADR_QM5_1017.json` document that G4 can read to accept zero trades for scaffold runs (same mechanism used for other known-zero-trade scenarios)
