Root cause:

Friday-close exits used `CTrade.PositionClose(ticket)` through the framework close-all path. Close deals can arrive with `DEAL_MAGIC=0`, so `QM_FrameworkOnTradeTransaction` rejected those exit deals and the q08 stream missed them. The missing F3 deals were close events with blank comments, mostly Friday broker-session closes.

Implemented fix:
- Added `QM_FrameworkExitReasonFromText()` to classify framework close reasons.
- Changed framework close-all paths to use `QM_TM_ClosePosition(ticket, reason)` instead of raw `CTrade.PositionClose(ticket)`.
- This preserves the position magic on the close request and records the intended exit reason.
- Added explicit default strategy session inputs to QM5_10094 setfiles to satisfy build guardrails without changing behavior.
