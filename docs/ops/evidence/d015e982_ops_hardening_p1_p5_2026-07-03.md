### Part 1 — Scaled Phase Timeouts (farmctl.py)

Added `_WORKLOAD_SCALED_PHASES` frozenset (`Q05`, `Q06`, `Q07`) and
`_scale_timeout_for_workload(base_min, payload, phase)` that scales the
base timeout by `date_range_years / 8.0` × `seeds / 3.0`, capped at `4×`.
`_active_timeout_min_for_work_item()` calls the scaler for these phases
so a 14-year full-history Q07 does not timeout prematurely.

### Part 2 — Magic Resolver Loud Drops + compile_ea Guard

**update_magic_resolver.py**:
- `load_rows()` now returns `(rows, dropped_ea_ids)` instead of just `rows`.
- If any ea_ids in magic_numbers.csv are dropped because their EA dir is
  missing under framework/EAs/, a LOUD `[WARNING]` block with the full list
  is printed to stderr.
- `--strict` flag causes exit 2 if any rows were dropped. Intended for CI /
  pump pre-flight calls to detect stale-resolver / missing-dir mismatches.

**compile_ea.py**:
- Added `RESOLVER_MQH` constant and `magic_in_resolver(ea_id)` function that
  reads the `QM_MAGIC_REG_EA_ID` array from `QM_MagicResolver.mqh` and checks
  whether `ea_id` is baked in.
- New pre-compile gate after `MAGIC_NOT_REGISTERED`: if the ea_id is in the
  CSV but **not** in the baked MQH, returns verdict `MAGIC_NOT_IN_RESOLVER`.
  This catches the case where the resolver is stale (not regenerated since the
  EA was registered/dir created) — a compile against a stale resolver produces
  a non-functional .ex5 where `QM_MagicRegistered()` always returns false.

### Part 3 — Seasonal Full-Year Prescreen Window (farmctl.py)

At dispatch time, if `strategy_type_flags` contains `calendar`, `season`, or
`month`, the prescreen date range is extended to the full calendar year
(`{to_year}.01.01` → `{to_year}.12.31`) instead of the default H2-only window.
This prevents seasonal strategies from being screened on a 6-month window that
misses Apr–Jun seasons.
