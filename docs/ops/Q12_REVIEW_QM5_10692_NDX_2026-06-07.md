## 8. Portfolio-depth recovery (post-decision, 2026-06-07)

OWNER chose to build a diversified book around this sleeve. Investigating the path to
member #2 revealed the real blocker was **not** strategy quality but a parser bug:

- The 3 nearest candidates sat at Q09 `NEED_MORE_DATA` with `portfolio_trade_count_below_min`
  on **phantom 0-trade readings**. Their Q08 ran on a German-locale terminal (T2/T6); the
  English-only `Get-ReportMetricValue` missed the German labels → `REPORT_FORMAT_DRIFT` →
  recorded 0 trades.
- Proof: `QM5_10440/NDX` actually traded **451×, net +$63,872, PF 1.22, max equity DD 14.4%,
  83% real ticks** — read directly from its German `report.htm`.

**Fixed:** `Get-ReportMetricValue` is now alias-aware (English → German), commit `2df91ef83`,
validated against the real report. **Re-queued** Q08 for QM5_10440 (NDX), QM5_10513 (XAUUSD),
QM5_10069 (XAUUSD) — pending, will now grade on real trades.

**Diversity caveat:** even if all three admit, the book is 2× NDX + 2× XAUUSD. A genuine
diversified portfolio needs more assets/mechanisms (FX majors, carry, mean-reversion,
seasonal) — the `strategy_priority` diversification weighting drives this over time.

**Systemic note:** this German-locale false-fail is not unique to these three — it has been
silently classifying genuine EAs as INFRA/zero-trade across the funnel whenever a run landed
on T2/T6. The parser fix recovers that for all *future* runs; a sweep for past T2/T6
false-fails (and/or the ops fix of setting T2+T6 UI to English) would recover more.
