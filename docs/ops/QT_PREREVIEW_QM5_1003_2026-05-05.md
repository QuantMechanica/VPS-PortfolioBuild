## NOTE — Dynamic stop widening from entry price (lines 159–169)

**Observation (non-blocking, verify against card §7):**

`Strategy_ManageOpenPosition()` recomputes the stop each bar as `QM_StopRulesStopFromDistance(_Symbol, side, price_open, stop_distance)` — where `price_open` is the position's open price (not the current price). If ATR increases after entry, the stop distance widens, moving the stop FURTHER from the current price (opposite direction of a trailing stop).

This can allow losses greater than the original entry-bar ATR stop if volatility spikes post-entry.

The card §7 says "no trailing/partial/BE; only protective stop logic is maintained." If the intent of "protective" is a static stop from entry, the code is correct — but the dynamic ATR adjustment can cause the stop to widen, which is non-standard for a "protective" stop.

**Action for formal QT review:** Confirm with Strategy Card §7 text whether the dynamic from-entry stop is explicitly specified, or whether a static stop at entry-bar ATR would be more faithful to the card.
