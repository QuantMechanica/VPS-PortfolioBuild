## 4. Diagnosis

**Not a zero-trade strategy failure at all. P2_infra_fail mislabelled as zero-trade.**
Every row shows `BARS_ZERO + HISTORY_CONTEXT_INVALID` — the tester started against a
period with no D1 bars loaded. Total trades = 0 because no bars were fed, not because
the signal was unproductive. The DL-062 zero-trade counter does not currently
distinguish "0 trades because no history" from "0 trades because signal never fired".
Recommending entry-condition relaxation here would be reasoning from corrupted data.

Two stacked defects:

1. **Enqueue used the wrong universe**: ~27 of 30 rows are on FX/commodities. Card
   declares US equity indices only. Same routing defect class as QM5_10020.
2. **Even on the right symbol (SP500.DWX/UK100/GDAXI), runs failed with NO_HISTORY.**
   This mirrors `[[project_qm_m1_history_gap_infra_fail]]` from memory, but on D1
   instead of M1, on the custom equity-index symbols. The window the tester chose
   has no D1 bars in the broker history for these custom symbols.

Also relevant from memory: `[[project_qm5_1044_perf_rework_2026-05-16]]` flags a
per-tick full-EMA recompute making the EA too slow. That is orthogonal and still open;
neither this rework nor the infra fix unblocks it.
