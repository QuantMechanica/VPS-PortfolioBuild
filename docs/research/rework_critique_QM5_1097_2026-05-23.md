# QM5_1097 unger-gold-intraday-bias — zero-trade rework critique

Router fired DL-062 on `completed=37 / fail=36 / zero_trade=36 (zt_pct=1.0)`.
The card declares **XAUUSD.DWX only** with M15 execution. The classifier's 100%
zero-trade ratio is the same dispatcher universe-mismatch pattern as the four
prior critiques — but there is also a real signal worth investigating on
the **one** in-universe run: XAUUSD M15 with the actual P2 setfile produced
zero trades, suggesting the setfile is not carrying the bias-slot list the
EA needs to fire.

runs produced exactly 0 trades over the full 2024 calendar year**.

## 2. Two root causes — one trivial scope issue, one real setfile concern

### 2a. Dispatcher universe mismatch (scope, trivial)

Card universe is the single line "Universe: XAUUSD.DWX". P2 enqueue fanned
across 36 DWX FX/index symbols anyway. 36/37 zero-trade fails are on symbols
that the strategy by construction cannot trade — the bias rule is hard-coded
to gold session-time bias and computes its slot threshold against
`ATR(14, H1)` of gold; running it on AUDCAD or NZDJPY is non-sensical.

This is the same dispatcher bug as QM5_1088 / QM5_1089 / QM5_1096 / QM5_1048
/ QM5_10020 (memory `project_qm_dispatcher_universe_mismatch_2026-05-23`).

### 2b. Setfile-side bias-slot configuration likely empty (real)

The card §Mechanik is explicit on this point:

> "For the first implementation, slot list is a fixed input selected in
> setfile, not recalculated live. The historical calculation above is the
> research rationale and P3 sweep driver."

If the P2 baseline setfile for XAUUSD.DWX was generated by
`framework/scripts/gen_setfile.ps1` without populating the seed slots
(`LONG 08:00-09:00`, `SHORT 14:00-15:00`), then the EA has **no slot at
which to fire** and zero trades is the correct, deterministic output of
its own rules — not edge failure.

This is the most likely explanation for the XAUUSD M15 run producing 0
trades despite:
- Clean tester runs (run_01 + run_02 both `status: OK`, `oninit_failure: false`).
- model=4 real-ticks confirmed.
- 2024 had multiple high-vol gold sessions where the ATR-vol-floor filter
  (skip if H1 ATR < 60% of 60-day median H1 ATR) would have passed easily.
- Per the card seed, two H1 slots/day × ~250 trading days × 0.5 filter
  pass-through × `one trade per symbol per day` cap ≈ 100–250 candidate
  entries per year — but only if the setfile actually carries the slot
  inputs.

I do not have the setfile contents in this critique window. The
**verification step** in §6 is to read
`C:\QM\repo\framework\EAs\QM5_1097_unger-gold-intraday-bias\sets\QM5_1097_unger-gold-intraday-bias_XAUUSD.DWX_M15_backtest.set`
and confirm whether the slot-list inputs are populated. If they are empty
or default-zero, this is a setfile-generation bug, not a strategy bug.

### 2c. min_trades_required=150 on FX fan-out (orthogonal)

Even ignoring 2a/2b: `expected_trades_per_year_per_symbol: 50` × 1 year × 3
(typical safety factor) = 150, which is what the dispatcher used. That's a
reasonable floor for the **gold** target but applying it to FX (where the
strategy fires zero by construction) is doubly inappropriate.

## 3. Why the DL-062 trigger fired

100% zero-trade fingerprint is the strongest possible signal to the
classifier, but it is also the most easily produced by:
- universe-mismatch fan-out (every out-of-universe symbol gives 0 trades);
- empty/default setfile inputs (no slots configured → no entries possible);
- not edge mortality (the gold M15 bias edge cannot fail differently from
  a setfile-empty EA in this telemetry).

## 4. Recommended change vector

Reject the router hint to relax entry conditions / substitute signal logic
before the setfile-generation question is settled. Required actions:

1. **Verify setfile contents (immediate)**: read the P2 baseline setfile for
   XAUUSD.DWX M15. Confirm whether the seed slot inputs (08:00 long, 14:00
   short) are present. If they are missing, this is a `gen_setfile.ps1` /
   build-side gap — not strategy mortality.

2. **Ops (codex)**: honor `target_symbols` from the card. For a single-symbol
   strategy with a universe of `[XAUUSD.DWX]`, the dispatcher should not
   create 36 out-of-universe work items. Same fix shared with QM5_1088 /
   QM5_1089 / QM5_1096 / QM5_1048 / QM5_10020.

3. **Setfile schema (codex)**: bias-system strategies (calendar/time-slot)
   need a setfile-side schema for slot-list inputs. Card declares
   "P3 parameter grid" — the P2 baseline should ship with the card's seed
   slots populated, not an empty grid placeholder.

4. **Re-enqueue (after #1 + #3)**: P2 on XAUUSD.DWX only, M15 execution,
   H1 bias slot selection per card. ≥5y window (the bias is by-construction
   stationary on the trading-day calendar; multi-year is needed for the
   skip filters to have meaningful sample). `min_trades_required = 0.4 ×
   expected × years = 0.4 × 50 × 5 = 100` across the window.

5. **Edge Lab compliance**: card pre-dates the 2026-05-22 charter. Live
   `RISK_PERCENT = 0.25%` is conservative (charter ≤5% daily / ≤10% total
   DD compatible). News filter declared ("Skip high-impact USD news +/-30
   min") satisfies the charter's mandatory news blackout. Friday-close
   protection declared. Mechanical, no-ML, no grid/martingale — all
   satisfied. ATR-stop = 1.0 × ATR(14,H1) — bounded per-trade DD.

6. **Do NOT mark DEAD.** Do NOT shorten the H1-ATR vol-floor or news skip
   to "produce more trades" before the setfile-emptiness hypothesis is
   ruled out.

## 5. Falsification — when this critique becomes wrong

If, after the setfile is verified to carry the seed slots and re-enqueue
on XAUUSD.DWX only at M15 over ≥5y still produces fewer than 100 total
trades, then the critique is wrong and either:
- (a) The H1 ATR vol-floor (skip if `ATR(14,H1) < 60% × 60d_median`) is
  too strict for the post-2020 XAUUSD vol regime — instrument and
  recalibrate via a separate decision;
- (b) The 08:00 / 14:00 seed slots no longer encode an active bias on
  XAUUSD (the bias has decayed). This would be a legitimate kill verdict.

Current evidence does not yet support either.
