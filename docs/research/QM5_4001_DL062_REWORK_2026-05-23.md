## Root-Cause Analysis

### ONINIT_FAILED — Universal EA Initialization Failure

The EA's `OnInit()` function fails before a single bar is processed. This means:
- No trades are possible (zero-trade is a consequence, not the root cause)
- The strategy's signal logic is never reached
- The DL-062 zero-trade classification is correct in effect but misleading in cause

`ONINIT_FAILED` in the QM5 framework typically results from:
1. Missing or invalid input parameters required by `OnInit()`
2. An indicator handle creation failure (`iMA`, `iRSI`, `iATR` returns INVALID_HANDLE)
3. A required symbol/period resource that cannot be loaded in the tester

### Card Incompleteness — Probable Build Defect Source

The QM5_4001 card is severely underspecified. Critical gaps that would cause an
implementation to fail at initialization:

| Required Element | Card Status | Impact |
|-----------------|-------------|--------|
| Timeframe | **Missing** — no period specified | The EA builder likely defaulted to a timeframe that conflicts with the scoring indicators or produces invalid ATR handles |
| Short-side entry | **Missing** — only long Score≥3 defined | Incomplete EA structure may cause init failure if the framework requires both sides |
| Stop Loss | **Missing** — no SL distance or ATR multiple | Framework risk-parameter validators may reject the EA at init |
| Position sizing | **Missing** — no RISK_FIXED or RISK_PERCENT | Required framework input; absence causes init failure |
| Symbol whitelist / target_symbols | **Missing** | Dispatcher ran 10 symbols from a default basket |
| expected_trades_per_year_per_symbol | **Missing** | min_trades floor defaults to an arbitrary value |

The full card body is:
> Long if Score >= 3 (Factors: SMA(50)>SMA(200), RSI<40, ATR > MA(ATR), BarColor=Green). Exit on Score <= 1.

This is a concept sketch, not a buildable pipeline specification.

### Strategy Concept Assessment

The multi-factor scoring concept is legitimate (buying pullbacks in uptrends with
expanding volatility and a reversal candle). However, the specific conditions have
problems beyond the build defect:

**Long-only limitation:** RSI<40 (oversold) + SMA50>SMA200 (uptrend) + ATR>MA(ATR)
(expanding volatility) + BarColor=Green requires a pullback reversal scenario. Without
a symmetric short side, the EA misses half the market.

**RSI<40 in an uptrend is rare:** In trending markets (SMA50>SMA200), RSI rarely
drops below 40. Score≥3 of 4 factors requires at least RSI<40 or another factor, but
the logical combination of uptrend + strong oversold RSI is genuinely uncommon.

**Timeframe dependency:** "SMA(50)" means very different things on M1 (50 minutes)
vs. H4 (50 bars = ~8.3 days) vs. D1 (50 bars = 10 weeks). The strategy is undefined
without a timeframe.

**No falsification criterion:** The card has no falsification section. An EA that
exits when Score≤1 with no hard stop has unlimited adverse excursion potential.

## Verdict

**RECYCLE.** This card needs a complete rework before any EA build is attempted.

### Minimum Requirements for Rework

1. **Timeframe:** Specify H4 or D1 (D1 is more consistent with the SMA-crossover
   macro-filter / RSI-pullback timing at swing scale).

2. **Short side:** Add symmetric short entry: Score≥3 where factors are
   SMA(50)<SMA(200), RSI>60, ATR>MA(ATR), BarColor=Red.

3. **Stop loss:** ATR-based hard stop, e.g., `SL = 1.5 × ATR(14, H4)` from entry.

4. **Position sizing:** `RISK_FIXED = 1000` for P2 backtest; `RISK_PERCENT = 0.5` for
   live. Add to frontmatter.

5. **Target symbols:** Define explicit whitelist (e.g., liquid FX majors + XAUUSD
   for H4, or broader universe for D1 weekly rotation).

6. **expected_trades:** Derive from the timeframe + filter logic. On D1, SMA-cross
   + RSI<40 + expanding ATR + green bar should yield ~15–30 entries/year/symbol.

7. **Falsification:** Add "if long-Score≥3 entries are not profitable in trending
   regimes but ARE profitable in mean-reverting regimes, the thesis is inverted —
   kill the strategy."

### Scoring System Design Note

If the intent is a "3 out of 4 factors" scoring model, consider making the factor
weights explicit: a genuine scoring EA should rank symbols or time entries, not just
gate on a binary count. As written, Score≥3 is effectively a 3-of-4 logical AND with
RSI<40 being the most restrictive and likely always required (it's the only non-trend
factor). Consider whether RSI<40 should be a mandatory gate rather than a scoring
factor, with the other 3 as confirmation.
