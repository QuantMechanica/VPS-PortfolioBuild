## Defect Analysis

### Defect 1: INFRA_FAIL — summary_missing

Five whitelist symbols (EURUSD, GBPUSD, AUDUSD, GDAXI, UK100) ran the smoke year (MT5
exit 0) but did not produce a summary report after 2 retries. NDX.DWX completed
successfully with a summary (MIN_TRADES_NOT_MET verdict), which rules out a general
compilation failure.

Likely cause: the Modified-Schiff pitchfork implementation uses dual-timeframe arrays
(H4 main chart + D1 secondary) and/or dynamic ATR-based geometry that triggers a
framework report-parsing failure under specific conditions on these symbols. Alternatively,
the EA completes zero trades AND triggers a report-path edge case not handled by the
smoke runner for those symbols (different from NDX where MIN_TRADES_NOT_MET is the
explicit path).

**Action required:** OPS_FIX — investigate summary_missing on EURUSD/GBPUSD/AUDUSD/
GDAXI/UK100. Compare successful NDX.DWX report output path vs. failed symbols. Check
if the EA's OnDeinit report-writing is symbol-dependent.

### Defect 2: Dispatcher Universe Mismatch

The card defines a specific whitelist of 11 symbols (EURUSD, GBPUSD, USDJPY, AUDUSD,
USDCAD, USDCHF, XAUUSD, NDX.DWX, WS30.DWX, GDAXI.DWX, UK100.DWX). The dispatcher
created work items for the full DWX universe (~37 symbols), including AUDCAD, AUDCHF,
AUDJPY, AUDNZD, CADCHF, CADJPY, CHFJPY, EURAUD, EURCAD, EURCHF, EURGBP, EURJPY,
EURNZD, GBPAUD, GBPCAD, GBPCHF, GBPJPY, GBPNZD, NZDCAD, NZDCHF, NZDJPY, NZDUSD.

These off-whitelist pairs either have no set file or will produce zero trades when run,
because the EA's signal logic is calibrated for liquid FX majors + indices + gold only
(pivot detection parameters were designed for these instruments).

**Action required:** Cancel pending off-whitelist work items. OPS_FIX to enforce
card.target_symbols at enqueue time (recurring DL-062 false-positive class).

### Defect 3: MIN_TRADES_NOT_MET on NDX.DWX and USDCAD.DWX

Both are valid whitelist symbols. The backtests ran (smoke 2018-2022, 1-year smoke)
but produced fewer than 10 trades. The Modified-Schiff entry conditions stack heavily:

- 3-pivot ABC structure within last 100 H4 bars (depth=12, dev=5, backstep=3)
- Pivot-magnitude gate: `|P0-P1| >= 2.0 × ATR(14,D1)` AND `|P2-P1| >= 1.5 × ATR(14,D1)`
- Pitchfork freshness: P2 within last 50 H4 bars
- Price at warning-line (within 0.2× ATR(H4))
- Rejection bar: body-ratio ≥ 0.40, close back above/below WL
- Macro-bias check: price in correct half of pivot range
- Spread guard + session gate (07:00–21:00 broker time)
- No re-entry on same pivot triple within 12 bars

On NDX.DWX (high-volatility index), the ATR(D1)-based magnitude gate is proportionally
larger in price terms, which may filter many valid pitchfork setups. On USDCAD.DWX
(lower-volatility, commodity-linked pair), pivot-swing amplitude may fall below the
magnitude threshold frequently.

The card states expected_trades_per_year_per_symbol=20. If genuine frequency on NDX +
USDCAD is below the min_trades=10 threshold, this is not a zero-trade problem per se —
it may reflect honest low-frequency signal generation on these specific symbols.

**Assessment:** These two failures are informative but secondary. Until INFRA_FAILs are
resolved for the core FX pairs, cannot determine if the strategy has a structural
zero-trade problem or merely a low-frequency issue on 2 of 11 whitelist symbols.
