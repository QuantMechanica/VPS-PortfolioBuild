## 5. v0 filter rules applied to this source

- **Mechanical only.** The FSB-generated EA is mechanical by construction. Verifiable from `.mq5` source — entries fire on indicator-combination signals from the 5 slots; no discretionary intervention path is exposed.
- **No Machine Learning.** No ONNX, no neural net, no online learning. PASS for `EA_ML_FORBIDDEN`. R4-clean on the ML dimension.
- **`.DWX` suffix discipline.** Course uses raw symbol names (AUDNZD, EURAUD, etc.). V5 deployment maps to `.DWX` suffixes. Per-card § 11 flags `dwx_suffix_discipline` (mechanical mapping, not a true risk).
- **`scalping_p5b_latency` — BINDS.** M1 timeframe with M30 filter is scalping-class. Cards from this source MUST be flagged for P5b VPS-realistic-latency calibration. SL=600 / TP=900 points (60 pip / 90 pip on 5-digit broker) is wider than HFT but still latency-sensitive.
- **`grid_1pct_cap` — POSSIBLY BINDS.** The `Martingale_Multiplier` parameter defaults to 1 (off), but the path exists. Cards extracted from this source must EXPLICITLY confirm `Martingale_Multiplier=1` in the recommended setfile AND any setup that engages `Martingale_Multiplier > 1` is R4-FAIL and must be dropped at card-synthesis time.
- **`one_position_per_magic_symbol`.** The EA has `Adding_Amount` and `Reducing_Amount` parameters which suggest position-scaling. Per-card § 11 must confirm whether scaling-in / scaling-out paths violate V5's 1-pos-per-magic rule, or whether they only adjust the single open position size.
- **`risk_mode_dual`.** Course teaches "scale-up money management" (PDF + module 6). V5 framework supports RISK_FIXED (backtest) / RISK_PERCENT (live). If the scaling-up logic is account-equity-percent-based (rather than PnL-based), it maps cleanly. If it's drawdown-recovery martingale-like, it's R4-FAIL.
- **`news_pause_default`.** Module 11 video 3 "Trading During The News" — author teaches active trading during news. Cards must explicitly reconcile against V5's news-blackout default (30 min before / 15 min after high-impact events).
- **`friday_close`.** M1 scalping should be intraday-close-compatible; expect no Friday-close exception needed.
